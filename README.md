# Neway RP — Discord-бот модерации

Бот для управления РП-проектом **Neway RP**. Работает только на сервере
с ID `1520321736611070013` — на любом другом сервере команды просто не
будут реагировать (как если бы бота там не было).

Все команды начинаются с префикса `-`.

---

## Что умеет бот

### Автовыдача роли новым участникам
- При входе любого нового пользователя на сервер ему автоматически
  выдаётся роль `1521077456679731200`.
- Никаких команд для этого вызывать не нужно — срабатывает само по
  событию входа на сервер.
- Если выдать роль не получится (например, не хватит прав), в консоли
  бота появится сообщение об ошибке (см. «⚠️ Важно про роль бота»
  ниже).

### `-report @пользователь`
*Доступно всем участникам сервера.*
- Собирает по всему серверу последние сообщения упомянутого
  пользователя (лимит — `REPORT_MESSAGE_LIMIT = 20`, с кликабельными
  ссылками на каждое).
- Достаёт данные профиля: дата создания аккаунта, дата входа на
  сервер, текущие роли.
- Отправляет оформленный отчёт в канал `1521050858719154226`.
- Автору команды приходит подтверждение об успешной отправке.
- **Кулдаун:** 60 сек, 7 сек для привилегированных ролей (см. раздел
  «Кулдауны» ниже) — общий с `-cancel`.

### `-license @пользователь <1|2> <ID канала-форума>`
*Доступно ролям из `LICENSE_ALLOWED_ROLE_IDS`.*

Выдаёт лицензию участнику РП и открывает ему доступ к посту-форуму.

⚠️ **Важно:** канал, который вы указываете, должен быть каналом
**типа «Форум»** (Discord Forum Channel), а не обычным текстовым
каналом. Если указать ID текстового/голосового канала — бот откажет
с ошибкой «Неверный тип канала». Это сознательное ограничение: лицензии
Neway RP привязаны к форумам анкет/заявок, а не к обычным чатам.

- **Тип 1 — Участник РП** (`RPP-XXXXXXX`)
  - Выдаёт роль `1520321736631914679`.
  - На форуме: можно создавать новые посты (`create_public_threads`) и
    отвечать в постах (`send_messages_in_threads`), читать историю,
    прикладывать файлы, ставить реакции — но **без** управления
    сообщениями/тредами.
- **Тип 2 — Менеджер РП** (`RPM-XXXXXXX`)
  - Выдаёт роль `1520321736631914678`.
  - То же самое, **плюс** `manage_messages` и `manage_threads`
    (можно удалять/закреплять чужие сообщения и посты).

**Ограничения:**
- Указанный форум должен находиться в категории `LICENSE_CATEGORY_ID`
  (`1520456833599279205`). Форум из другой категории — отказ.
- Один участник может получить несколько лицензий одного типа на
  разные форумы — каждая фиксируется отдельной записью в базе с своим
  ID (`RPP-0000001`, `RPP-0000002`, …).
- Лог о выдаче уходит в `1521051946675601508`.

**Пример:**
```
-license @username 1 1234567890123456789
```
(где `1234567890123456789` — ID канала-форума, не текстового канала)

### `-cancel @пользователь <1|2>`
*Доступно ролям из `CANCEL_ALLOWED_ROLE_IDS`.*
- Отзывает **все** активные лицензии указанного типа у пользователя.
- Снимает соответствующую роль лицензии.
- Снимает выданные права (overwrite) со всех форумов, на которые
  действовали отозванные лицензии.
- Помечает записи в базе данных как отозванные (с указанием, кто и
  когда отозвал).
- Лог уходит в `1521052020633505912`.
- **Кулдаун:** общий с `-report` (60 сек / 7 сек для привилегированных
  ролей).

**Пример:**
```
-cancel @username 1
```

### `-data [запрос]`
*Доступно ролям из `DATA_ALLOWED_ROLE_IDS`.*

Поиск по базе данных лицензий. Тип поиска определяется автоматически
по формату запроса:

| Запрос | Что ищет |
|---|---|
| `RPP-0000001` / `RPM-0000003` | Точное совпадение по ID лицензии |
| `@пользователь` или числовой Discord ID | Все лицензии этого пользователя |
| Любая другая строка | Частичное совпадение по никнейму (без учёта регистра) |

Выводит по каждому найденному пользователю: все его лицензии
(активные 🟢 / отозванные 🔴), даты выдачи и отзыва, статус канала
(существует или удалён), и заметку персонала (видна только ролям с
доступом к `-data`).

**Примеры:**
```
-data RPP-0000001
-data @username
-data 123456789012345678
-data username
```

### `-note @пользователь <текст>`
*Доступно ролям из `DATA_ALLOWED_ROLE_IDS`.*
- Создаёт или **полностью перезаписывает** заметку о пользователе в
  базе данных.
- Заметка видна только через `-data` тем же ролям — обычные участники
  её не видят.

**Пример:**
```
-note @username Подозрительное поведение, следить за активностью.
```

### `-del <количество>`
*Доступно ролям из `DELETE_ALLOWED_ROLE_IDS`.*
- Удаляет указанное количество последних сообщений **в том канале,
  где вызвана команда**.
- Сообщение с самой командой удаляется автоматически, ответ бота
  самоудаляется через 8 секунд.
- **Лимит:** до `DEL_MAX_MESSAGES` (1000) сообщений за вызов.
- **Кулдаун:** `DEL_COOLDOWN_SECONDS` (5 сек), фиксированный, отдельный
  от кулдауна `-report`/`-cancel`.
- Лог уходит в `1521076264222588958`.

**Пример:**
```
-del 50
```

### `-delus @пользователь <количество>`
*Доступно тем же ролям, что и `-del`.*
- Удаляет последние N сообщений указанного пользователя **по всем
  каналам сервера**, к которым у бота есть доступ.
- Сканирует до `DELUS_SCAN_LIMIT_PER_CHANNEL` (2000) последних
  сообщений в каждом канале при поиске.
- **Лимит:** до `DELUS_MAX_MESSAGES` (500) сообщений за вызов.
- **Кулдаун:** `DELUS_COOLDOWN_SECONDS` (10 сек), фиксированный.
- Лог (с разбивкой по каналам) уходит в `1521076264222588958`.
- ⚠️ Discord не позволяет массово удалять сообщения старше **14 дней**
  одним запросом — для таких сообщений бот автоматически переключается
  на удаление по одному, что заметно медленнее. Это ограничение
  Discord API, не ошибка бота.

**Пример:**
```
-delus @username 100
```

### `-help`
*Доступно всем, но показывает только те команды, к которым у вызвавшего
есть доступ.*
- Если у вас нет ни одной из ролей, открывающих доступ к команде,
  блок с её описанием в `-help` просто не появится — это ожидаемое
  поведение, а не баг. Например, `-license` будет виден в `-help`
  только тем, у кого есть одна из ролей из `LICENSE_ALLOWED_ROLE_IDS`.
- Если кажется, что какой-то команды «нет» в `-help` — сначала
  проверьте, есть ли у вас соответствующая роль доступа в `config.py`.

### Кулдауны
| Команда | Длительность | Зависит от ролей? |
|---|---|---|
| `-report` / `-cancel` | 60 сек (общий таймер на двоих) | Да — 7 сек для ролей из `SHORT_COOLDOWN_ROLE_IDS` |
| `-del` | 5 сек | Нет, фиксированный |
| `-delus` | 10 сек | Нет, фиксированный |

`-license`, `-data`, `-note`, `-help` не имеют отдельного кулдауна —
доступ к ним ограничен только ролями.

## Структура проекта

```
neway-bot/
├── bot.py             — основной код бота (события и все команды)
├── config.py          — все ID, цвета, тексты — настройки в одном месте
├── cooldown.py         — система кулдаунов с разным временем по ролям
├── license_db.py       — SQLite-база лицензий и заметок (license_data.db)
├── requirements.txt    — список Python-зависимостей
├── .env.example        — пример файла с токеном (скопировать в .env)
└── README.md           — этот файл
```

Если когда-нибудь нужно поменять ID канала, роли, цвета или лимиты —
почти всё лежит в `config.py`, копаться в основном коде `bot.py` не
придётся.

## Шаг 1. Создание бота в Discord Developer Portal

1. Перейдите на https://discord.com/developers/applications
2. Нажмите **New Application**, дайте имя (например, `Neway RP Bot`).
3. В разделе **Bot** (слева в меню):
   - Нажмите **Reset Token** (или **Add Bot**, если бота ещё нет) и
     **скопируйте токен** — он понадобится дальше. Никому его не
     показывайте и не публикуйте.
   - Включите два переключателя в разделе **Privileged Gateway
     Intents**:
     - ✅ **SERVER MEMBERS INTENT** (нужен и для авторолей новым
       участникам, и для проверки ролей в командах)
     - ✅ **MESSAGE CONTENT INTENT**
     (без них бот не увидит ни роли участников, ни текст команд, ни
     момент входа нового пользователя на сервер).
4. В разделе **OAuth2 → URL Generator**:
   - Scopes: отметьте `bot`.
   - Bot Permissions: отметьте минимально необходимое:
     - `View Channels`
     - `Send Messages`
     - `Send Messages in Threads` (нужно для постов в форумах — `-license`)
     - `Create Public Threads` (нужно, чтобы лицензированные участники
       могли создавать посты в форуме — это разрешение выдаёт сам бот
       через overwrite, но у бота самого должно быть это право)
     - `Embed Links`
     - `Read Message History`
     - `Manage Roles` (нужно для выдачи/снятия ролей в `-license`/`-cancel`)
     - `Manage Channels` (нужно, чтобы бот мог менять права доступа к
       каналу-форуму в `-license`/`-cancel`)
     - `Manage Messages` (нужно для удаления сообщений в `-del`/`-delus`
       и для управления чужими сообщениями в форумах при лицензии типа 2)
     - `Manage Threads` (нужно для лицензии типа 2 — управление постами форума)
   - Скопируйте получившуюся ссылку внизу страницы и откройте её в
     браузере — появится окно приглашения бота на сервер. Выберите
     ваш сервер Neway RP и подтвердите.

### ⚠️ Важно про роль бота и `Manage Roles`
Discord не даёт боту выдавать или снимать роли, которые расположены
**выше** роли бота в списке ролей сервера (Настройки сервера → Роли).
Это касается автовыдачи роли новым участникам и выдачи/снятия ролей
в `-license`/`-cancel`. Поэтому:
- Зайдите в **Настройки сервера → Роли**.
- Найдите роль бота (она появится после приглашения, обычно называется
  как имя бота).
- **Перетащите её выше** ролей `1520321736631914679`,
  `1520321736631914678` (лицензии) и `1521077456679731200`
  (авторолья новым участникам).

Если это не сделать, бот будет писать в консоль ошибку «не хватает
прав» при попытке выдать/снять роль.

### ⚠️ Важно про канал-форум для `-license`
Команда `-license` принимает **только** ID канала типа «Форум»
(Forum Channel в Discord). Чтобы узнать ID нужного форума:
1. Включите режим разработчика: **Настройки пользователя → Расширенные
   → Режим разработчика**.
2. Кликните правой кнопкой по нужному форуму → **Копировать ID канала**.

Убедитесь также, что этот форум находится внутри категории с ID
`1520456833599279205` (`LICENSE_CATEGORY_ID` в `config.py`) — иначе
команда откажет с ошибкой «Канал не в разрешённой категории».

## Шаг 2. Установка зависимостей

Нужен **Python 3.10+** (рекомендуется 3.11/3.12).

```bash
# Создаём виртуальное окружение (необязательно, но рекомендуется)
python3 -m venv venv
source venv/bin/activate        # для Windows: venv\Scripts\activate

# Устанавливаем зависимости
pip install -r requirements.txt
```

## Шаг 3. Указываем токен

Скопируйте `.env.example` в `.env`:

```bash
cp .env.example .env
```

Откройте `.env` и впишите токен, который вы скопировали в Discord
Developer Portal:

```
BOT_TOKEN=ваш_настоящий_токен_сюда
```

## Шаг 4. Запуск

```bash
python3 bot.py
```

При первом запуске автоматически создастся файл `license_data.db`
(SQLite) с таблицами лицензий и заметок — ничего настраивать
дополнительно не нужно.

В консоли должно появиться:
```
[OK] Бот запущен как Neway RP Bot#1234 (ID: ...)
[OK] Работает только на сервере: 1520321736611070013
```

Если видите это — всё работает. Идём на сервер Discord и пишем
`-help`, чтобы увидеть список доступных вам команд.

## Шаг 5. Бесплатный хостинг навсегда

Локальный запуск с компьютера годится только для тестов — как только
вы закроете терминал или выключите ПК, бот отключится. Чтобы бот
работал 24/7 бесплатно, рекомендую один из вариантов ниже.

### 🥇 Рекомендую: **Railway** *(но есть нюанс)*
Самый удобный по интерфейсу, но с 2023 года бесплатный тир урезан до
небольшого количества часов в месяц с привязкой карты для
подтверждения — не подходит под «навсегда бесплатно» без оговорок.

### 🥇 Лучший вариант под «бесплатно навсегда»: **VPS от Oracle Cloud Free Tier**
Oracle Cloud даёт **бессрочно бесплатные** виртуальные машины (Always
Free): 2 ARM-инстанса (по 1 OCPU / 6 GB RAM каждый, либо собранные в
один помощнее) или 2 микро-инстанса AMD. Этого более чем достаточно
для дискорд-бота, который даже не нагружает CPU.

**Почему это лучший вариант:**
- Реально бессрочно бесплатно (не триал на месяц).
- Полноценный Linux-сервер — бот работает 24/7 без ограничений по
  времени, "не засыпает", в отличие от многих бесплатных PaaS.
- Можно поставить `systemd`, чтобы бот автоматически перезапускался
  при сбое или перезагрузке сервера.

**Краткая инструкция:**
1. Регистрируетесь на https://www.oracle.com/cloud/free/ (нужна карта
   для верификации личности, но с бесплатного тира деньги не
   снимаются).
2. Создаёте инстанс (Compute → Instance), выбираете образ **Ubuntu**,
   shape — любой из **Always Free** (помечены явно).
3. Подключаетесь по SSH (Oracle даёт готовую команду подключения).
4. На сервере:
   ```bash
   sudo apt update && sudo apt install python3-pip python3-venv git -y
   git clone <ваш_репозиторий_с_ботом>   # или загрузите файлы через scp
   cd neway-bot
   python3 -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt
   cp .env.example .env
   nano .env   # вписываете токен, Ctrl+O сохранить, Ctrl+X выйти
   ```
5. Чтобы бот работал постоянно и сам перезапускался, создаём службу
   systemd:
   ```bash
   sudo nano /etc/systemd/system/neway-bot.service
   ```
   Вставляем:
   ```ini
   [Unit]
   Description=Neway RP Discord Bot
   After=network.target

   [Service]
   Type=simple
   User=ubuntu
   WorkingDirectory=/home/ubuntu/neway-bot
   ExecStart=/home/ubuntu/neway-bot/venv/bin/python3 bot.py
   Restart=always
   RestartSec=5

   [Install]
   WantedBy=multi-user.target
   ```
   Затем:
   ```bash
   sudo systemctl daemon-reload
   sudo systemctl enable neway-bot
   sudo systemctl start neway-bot
   sudo systemctl status neway-bot   # проверка, что всё запустилось
   ```
   Логи смотреть так: `sudo journalctl -u neway-bot -f`

### 🥈 Альтернатива для совсем новичков (проще, но с компромиссами)

Если возиться с VPS не хочется, можно взять бесплатный тир хостинга
**fly.io** — он даёт небольшую бесплатную квоту, которой хватает на
одного лёгкого дискорд-бота, но требует привязку карты и иногда
меняет условия бесплатного тира, поэтому не гарантирован «навсегда».

Либо **Replit** с настройкой Always On — там Always On платный, так
что бесплатный Replit без доплат держать бота 24/7 не сможет (будет
выключаться при простое).

**Вывод:** если нужно по-настоящему бесплатно и навсегда — берите
Oracle Cloud Free Tier с systemd, как описано выше. Это самый честный
и надёжный бесплатный вариант на сегодня.

## Частые проблемы

**Бот не реагирует ни на одну команду.**
Проверьте, что вы пишете команду именно на сервере с ID
`1520321736611070013` — на других серверах бот молчит специально.

**Бот не видит сообщения / не отвечает вообще.**
Убедитесь, что включён **MESSAGE CONTENT INTENT** в Developer Portal
(см. Шаг 1).

**В `-help` не вижу команду `-license` (или любую другую).**
`-help` показывает только те команды, к которым у вас есть доступ по
ролям. Если вашей роли нет в соответствующем списке в `config.py`
(`LICENSE_ALLOWED_ROLE_IDS`, `DATA_ALLOWED_ROLE_IDS` и т.д.) — блок
с этой командой просто не появится. Это не баг, а ожидаемое
поведение. Проверьте список ролей в `config.py` и при необходимости
добавьте туда нужную роль.

**`-license` отвечает «Неверный тип канала».**
Команда принимает только ID **канала-форума** (Forum Channel), а не
обычного текстового канала. Скопируйте ID именно форума (правый клик
по форуму → «Копировать ID канала»).

**`-license` отвечает «Канал не в разрешённой категории».**
Указанный форум должен находиться внутри категории с ID
`LICENSE_CATEGORY_ID` (см. `config.py`). Переместите форум в эту
категорию или поменяйте `LICENSE_CATEGORY_ID` на нужную.

**`-license`/`-cancel` говорят «не хватает прав» при попытке выдать/снять роль.**
Роль бота должна стоять **выше** соответствующих ролей лицензий в
списке ролей сервера. См. раздел «⚠️ Важно про роль бота» выше.

**`-license` не может выдать права на форум («Управление каналом»).**
У бота должно быть право `Manage Channels` на сервере (или хотя бы на
данной категории/форуме), иначе `set_permissions` завершится с
ошибкой `Forbidden`.

**`-report` находит мало сообщений или не находит вовсе.**
Бот может читать историю только в тех каналах, куда у него есть
доступ (`View Channels` + `Read Message History`). Если пользователь
писал только в канале, закрытом от бота, его сообщения туда не попадут.

## Если что-то нужно поменять

Все ID ролей/каналов, цвета, лимиты и тексты сообщений собраны в
`config.py` — большинство правок делается там, без изменения логики в
`bot.py`. Если нужно подправить что-то ещё (раздельные кулдауны на
команды, другой лимит сообщений, доп. роли с доступом и т.д.) — просто
скажите, поправим.
