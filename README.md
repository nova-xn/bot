# Neway RP — Discord Bot

Бот модерации и учёта РП-лицензий для сервера **Neway RP**. Ведёт базу
данных лицензий участников (выдача/отзыв с указанием ответственного
модератора), обрабатывает жалобы, чистит сообщения и защищён от двойного
выполнения команд при параллельных процессах.

**Версия:** `2.0.1 beta`
**Последнее обновление:** 02.07.2026
**Разработчик:** <@1148265088277545081>

## Возможности

| Команда | Доступ | Описание |
|---|---|---|
| `-report @user` | все | Отправляет жалобу на участника в канал администрации, прикладывая последние сообщения нарушителя. |
| `-license @user <1\|2> <channel_id>` | роли из `LICENSE_ALLOWED_ROLE_IDS` | Выдаёт лицензию (Участника РП / Менеджера РП), назначает роль и права на форум-канале. |
| `-cancel @user <1\|2>` | роли из `CANCEL_ALLOWED_ROLE_IDS` | Отзывает все активные лицензии указанного типа: снимает роль и права на всех связанных каналах. |
| `-data [запрос]` | роли из `DATA_ALLOWED_ROLE_IDS` | Поиск по базе лицензий — по ID лицензии, упоминанию, Discord ID или нику. Показывает статус, канал, кто выдал и кто отозвал. |
| `-note @user <текст>` | роли из `DATA_ALLOWED_ROLE_IDS` | Добавляет/перезаписывает приватную заметку к пользователю. |
| `-wipe @user [confirm]` | Администратор сервера | Безвозвратно удаляет всю историю лицензий пользователя из БД. |
| `-del <количество>` | роли из `DELETE_ALLOWED_ROLE_IDS` | Удаляет N последних сообщений в текущем канале. |
| `-delus @user <количество>` | роли из `DELETE_ALLOWED_ROLE_IDS` | Удаляет N последних сообщений пользователя по всем каналам сервера. |
| `-bot` | все | Показывает разработчика, версию и дату последнего обновления бота. |
| `-help` | все | Подробная справка — каждому показываются только доступные ему команды. |

Полные описания, ограничения и примеры для каждой команды — в `-help`
внутри Discord (тексты берутся из `_HELP_BLOCKS` в `bot.py`).

## Требования

- Python **3.10+** (используется синтаксис `int | None`)
- [discord.py](https://pypi.org/project/discord.py/)
- PostgreSQL (используется `license_db.py` для хранения лицензий,
  заметок и защиты от повторной обработки сообщений)
- `psycopg2` для подключения к PostgreSQL

Установка зависимостей:

```bash
pip install discord.py psycopg2-binary
```

Если в репозитории есть `requirements.txt` — используйте его:

```bash
pip install -r requirements.txt
```

## Настройка (`config.py`)

Бот ожидает объект `config` со следующими константами. Заполните
`config.py` под свой сервер перед запуском:

**Общее**
| Переменная | Назначение |
|---|---|
| `BOT_TOKEN` | токен бота (лучше брать из переменной окружения, не хранить в коде) |
| `PREFIX` | префикс команд (например, `-`) |
| `GUILD_ID` | ID сервера — бот работает **только** на нём |
| `FOOTER_TEXT` | текст в подвале всех embed-сообщений |

**Роли доступа к командам**
| Переменная | Назначение |
|---|---|
| `DATA_ALLOWED_ROLE_IDS` | кому доступны `-data` и `-note` |
| `LICENSE_ALLOWED_ROLE_IDS` | кому доступна `-license` |
| `CANCEL_ALLOWED_ROLE_IDS` | кому доступна `-cancel` |
| `DELETE_ALLOWED_ROLE_IDS` | кому доступны `-del` и `-delus` |

**Лицензии**
| Переменная | Назначение |
|---|---|
| `LICENSE_1_ROLE_ID` | роль, выдаваемая за лицензию типа 1 (Участник РП) |
| `LICENSE_2_ROLE_ID` | роль, выдаваемая за лицензию типа 2 (Менеджер РП) |
| `LICENSE_CATEGORY_ID` | категория, в которой обязаны находиться форум-каналы для выдачи лицензий |

**Автороли**
| Переменная | Назначение |
|---|---|
| `AUTOROLE_ON_JOIN_ID` | роль, выдаваемая новым участникам при входе на сервер |

**Каналы логов**
| Переменная | Назначение |
|---|---|
| `REPORT_LOG_CHANNEL_ID` | куда падают жалобы `-report` |
| `LICENSE_ISSUE_LOG_CHANNEL_ID` | лог выдачи лицензий |
| `LICENSE_REVOKE_LOG_CHANNEL_ID` | лог отзыва лицензий (`-wipe`) |
| `CANCEL_LOG_CHANNEL_ID` | лог `-cancel` |
| `DELETE_LOG_CHANNEL_ID` | лог `-del` / `-delus` |

**Лимиты и кулдауны**
| Переменная | Назначение |
|---|---|
| `REPORT_MESSAGE_LIMIT` | сколько последних сообщений прикладывать к жалобе |
| `DEL_MAX_MESSAGES` | максимум сообщений за один вызов `-del` |
| `DEL_COOLDOWN_SECONDS` | кулдаун `-del` |
| `DELUS_MAX_MESSAGES` | максимум сообщений за один вызов `-delus` |
| `DELUS_COOLDOWN_SECONDS` | кулдаун `-delus` |
| `DELUS_SCAN_LIMIT_PER_CHANNEL` | сколько сообщений сканировать на канал при `-delus` |

**Цвета embed'ов**
`COLOR_ERROR`, `COLOR_SUCCESS`, `COLOR_INFO`, `COLOR_REPORT`,
`COLOR_LICENSE`, `COLOR_CANCEL`, `COLOR_REVOKE`, `COLOR_DATA`,
`COLOR_DELETE` — HEX-цвета (`int`, например `0xFF0000`) для соответствующих типов сообщений.

## База данных

`license_db.py` хранит данные в PostgreSQL и отвечает за:

- выдачу/поиск/отзыв лицензий (`issue_license`, `revoke_licenses`,
  `search_by_license_id`, `search_by_discord_id`, `search_by_username`,
  `get_active_license_by_channel`) — каждая запись хранит, **кто выдал**
  (`issued_by`) и, если отозвана, **кто отозвал** (`revoked_by`);
- заметки персонала (`get_note`, `set_note`);
- полную очистку истории пользователя (`clear_history`, команда `-wipe`);
- защиту от двойного выполнения команд (`claim_message`,
  `cleanup_processed_messages`) — см. раздел ниже.

При старте бот сам вызывает `license_db.init_db()`, которая создаёт
нужные таблицы, если их ещё нет. Подключение к БД (хост/логин/пароль)
настраивается внутри `license_db.py` — обычно через переменные окружения.

## Защита от дублей

Если Discord повторно доставит одно и то же событие сообщения, или по
ошибке окажутся запущены два процесса бота с одним токеном одновременно
(например, при неудачном редеплое на хостинге), глобальная проверка
`globally_deduplicate_messages` в `bot.py` гарантирует, что тело команды
всё равно выполнится только один раз — `claim_message()` атомарно
«занимает» ID сообщения на уровне БД, так что защита работает даже между
разными процессами. Раз в 6 часов фоновая задача
`cleanup_processed_messages_loop` чистит старые записи из этой таблицы.

## Запуск

```bash
export BOT_TOKEN="ваш_токен_бота"
python bot.py
```

Бот подключается только к серверу с ID `config.GUILD_ID` — на любом
другом сервере глобальная проверка `globally_restrict_guild` заблокирует
выполнение всех команд.

## Лицензии участников

- Тип `1` — **Участник РП**, ID вида `RPP-0000001`
- Тип `2` — **Менеджер РП**, ID вида `RPM-0000001`
- На один канал одновременно может действовать только одна активная
  лицензия каждого типа.
- Полную историю (активные и отозванные лицензии) по любому участнику
  можно посмотреть через `-data`.
