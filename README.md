# Neway RP — Discord-бот модерации

Бот для управления РП-проектом **Neway RP**. Работает только на сервере
с ID `1520321736611070013`, на любом другом сервере команды просто не
будут реагировать (как если бы бота там не было).

## Что умеет бот

### Автовыдача роли новым участникам
- При входе любого нового пользователя на сервер ему автоматически
  выдаётся роль `1521077456679731200`.
- Никаких команд для этого вызывать не нужно — срабатывает само по
  событию входа на сервер.
- Если выдать роль не получится (например, не хватит прав), в консоли
  бота появится сообщение об ошибке — это поможет быстро понять
  причину (см. раздел «Важно про роль бота» ниже).

### `-report @пользователь`
- Доступно всем.
- Собирает по всему серверу последние 20 сообщений упомянутого
  пользователя (с кликабельными ссылками на каждое сообщение).
- Достаёт данные профиля: дата создания аккаунта, дата входа на
  сервер, текущие роли.
- Отправляет красиво оформленный отчёт в канал `1521050858719154226`.
- Автору команды приходит подтверждение об успешной отправке.

### `-cancel @пользователь`
- Доступно **только** участникам с одной из ролей:
  `1520321736673853512`, `1520321736673853511`, `1520321736673853510`,
  `1520321736673853504`, `1520321736673853503`, `1520321736648949936`,
  `1520321736648949935`, `1520321736648949932`.
- Если роли нет — бот ответит, что доступа нет, и больше ничего не
  сделает.
- Если роль есть — у упомянутого пользователя снимаются роли
  `1520321736631914679` и `1520321736631914678` (если они у него есть).
- Лог о выполнении (кто, кого, что сняли) отправляется в канал
  `1521052020633505912`.

### Кулдаун (КД)
- Общий КД на команды — **60 секунд** на пользователя.
- Если у пользователя есть роль `1520321736673853512`,
  `1520321736673853511` или `1520321736673853510` — КД всего
  **7 секунд**.
- КД общий для обеих команд (то есть `-report` и `-cancel` делят один
  таймер на пользователя — обычная практика анти-спама; если хотите
  раздельные таймеры на каждую команду, дайте знать, поправим).

### `-del количество`
Например: `-del 100`
- Удаляет указанное количество последних сообщений **в том канале,
  где вызвана команда**.
- Доступно только ролям: `1520321736673853512`, `1520321736673853511`,
  `1520321736673853510`, `1520321736673853505`.
- Лимит — **1000** сообщений за один вызов.
- Кулдаун — **5 секунд** (фиксированный, не зависит от ролей, отдельный
  от кулдауна `-report`/`-cancel`).
- Сообщение с самой командой удаляется автоматически, а ответ бота
  про результат самоудаляется через 8 секунд, чтобы не мусорить в чате.
- Лог о выполнении (кто, сколько запросил/удалено, в каком канале)
  отправляется в `1521076264222588958`.

### `-delus @пользователь количество`
Например: `-delus @user 100`
- Удаляет последние N сообщений **указанного пользователя по всем
  каналам сервера**, куда у бота есть доступ.
- Доступно только тем же ролям, что и `-del`.
- Лимит — **500** сообщений за один вызов.
- Кулдаун — **10 секунд** (фиксированный).
- Лог (кто удалил, у кого, сколько запросил/удалено, разбивка по
  каналам) отправляется в `1521076264222588958`.
- ⚠️ Discord не позволяет массово удалять сообщения старше **14 дней**
  одним запросом — для таких сообщений бот автоматически переключается
  на удаление по одному, что заметно медленнее. Это нормальное
  ограничение Discord API, не ошибка бота.

## Структура проекта

```
neway-bot/
├── bot.py             — основной код бота (события и команды)
├── config.py          — все ID, цвета, тексты — настройки в одном месте
├── cooldown.py        — система кулдаунов с разным временем по ролям
├── requirements.txt   — список Python-зависимостей
├── .env.example       — пример файла с токеном (надо скопировать в .env)
└── README.md          — этот файл
```

Если когда-нибудь нужно поменять ID канала, роли или текст — всё
лежит в `config.py`, копаться в основном коде `bot.py` не придётся.

## Шаг 1. Создание бота в Discord Developer Portal

1. Перейдите на https://discord.com/developers/applications
2. Нажмите **New Application**, дайте имя (например, `Neway RP Bot`).
3. В разделе **Bot** (слева в меню):
   - Нажмите **Reset Token** (или **Add Bot**, если бота ещё нет) и
     **скопируйте токен** — он понадобится дальше. Никому его не
     показывайте и не публикуйте.
   - Включите два переключателя в разделе **Privileged Gateway
     Intents**:
     - ✅ **SERVER MEMBERS INTENT** (нужен и для авторолей новым
       участникам, и для проверки ролей в командах)
     - ✅ **MESSAGE CONTENT INTENT**
     (без них бот не увидит ни роли участников, ни текст команд, ни
     момент входа нового пользователя на сервер).
4. В разделе **OAuth2 → URL Generator**:
   - Scopes: отметьте `bot`.
   - Bot Permissions: отметьте минимально необходимое:
     - `View Channels`
     - `Send Messages`
     - `Embed Links`
     - `Read Message History`
     - `Manage Roles` (нужно для снятия ролей в `-cancel`)
     - `Manage Messages` (нужно для удаления сообщений в `-del`/`-delus`)
   - Скопируйте получившуюся ссылку внизу страницы и откройте её в
     браузере — появится окно приглашения бота на сервер. Выберите
     ваш сервер Neway RP и подтвердите.

### ⚠️ Важно про роль бота и `Manage Roles`
Discord не даёт боту выдавать или снимать роли, которые расположены
**выше** роли бота в списке ролей сервера (Настройки сервера → Роли).
Это касается как автовыдачи роли новым участникам, так и снятия ролей
в `-cancel`. Поэтому:
- Зайдите в **Настройки сервера → Роли**.
- Найдите роль бота (она появится после приглашения, обычно называется
  как имя бота).
- **Перетащите её выше** ролей `1520321736631914679`,
  `1520321736631914678` (которые бот снимает в `-cancel`) и
  `1521077456679731200` (которую бот выдаёт новым участникам).

Если это не сделать, бот будет писать в консоль ошибку «не хватает
прав» при попытке выдать/снять роль.

## Шаг 2. Установка зависимостей

Нужен **Python 3.10+** (рекомендуется 3.11/3.12).

```bash
# Создаём виртуальное окружение (необязательно, но рекомендуется)
python3 -m venv venv
source venv/bin/activate        # для Windows: venv\Scripts\activate

# Устанавливаем зависимости
pip install -r requirements.txt
```

## Шаг 3. Указываем токен

Скопируйте `.env.example` в `.env`:

```bash
cp .env.example .env
```

Откройте `.env` и впишите токен, который вы скопировали в Discord
Developer Portal:

```
BOT_TOKEN=ваш_настоящий_токен_сюда
```

## Шаг 4. Запуск

```bash
python3 bot.py
```

В консоли должно появиться:
```
[OK] Бот запущен как Neway RP Bot#1234 (ID: ...)
[OK] Работает только на сервере: 1520321736611070013
```

Если видите это — всё работает. Идём на сервер Discord и пишем
`-report @кто-то`, чтобы проверить.

## Шаг 5. Бесплатный хостинг навсегда

Локальный запуск с компьютера годится только для тестов — как только
вы закроете терминал или выключите ПК, бот отключится. Чтобы бот
работал 24/7 бесплатно, рекомендую один из вариантов ниже.

### 🥇 Рекомендую: **Railway** *(но есть нюанс)*
Самый удобный по интерфейсу, но с 2023 года бесплатный тир урезан до
небольшого количества часов в месяц с привязкой карты для
подтверждения — не подходит под «навсегда бесплатно» без оговорок.

### 🥇 Лучший вариант под «бесплатно навсегда»: **VPS от Oracle Cloud Free Tier**
Oracle Cloud даёт **бессрочно бесплатные** виртуальные машины (Always
Free): 2 ARM-инстанса (по 1 OCPU / 6 GB RAM каждый, либо собранные в
один помощнее) или 2 микро-инстанса AMD. Этого более чем достаточно
для дискорд-бота, который даже не нагружает CPU.

**Почему это лучший вариант:**
- Реально бессрочно бесплатно (не триал на месяц).
- Полноценный Linux-сервер — бот работает 24/7 без ограничений по
  времени, "не засыпает", в отличие от многих бесплатных PaaS.
- Можно поставить `systemd`, чтобы бот автоматически перезапускался
  при сбое или перезагрузке сервера.

**Краткая инструкция:**
1. Регистрируетесь на https://www.oracle.com/cloud/free/ (нужна карта
   для верификации личности, но с бесплатного тира деньги не
   снимаются).
2. Создаёте инстанс (Compute → Instance), выбираете образ **Ubuntu**,
   shape — любой из **Always Free** (помечены явно).
3. Подключаетесь по SSH (Oracle даёт готовую команду подключения).
4. На сервере:
   ```bash
   sudo apt update && sudo apt install python3-pip python3-venv git -y
   git clone <ваш_репозиторий_с_ботом>   # или загрузите файлы через scp
   cd neway-bot
   python3 -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt
   cp .env.example .env
   nano .env   # вписываете токен, Ctrl+O сохранить, Ctrl+X выйти
   ```
5. Чтобы бот работал постоянно и сам перезапускался, создаём службу
   systemd:
   ```bash
   sudo nano /etc/systemd/system/nevay-bot.service
   ```
   Вставляем:
   ```ini
   [Unit]
   Description=Neway RP Discord Bot
   After=network.target

   [Service]
   Type=simple
   User=ubuntu
   WorkingDirectory=/home/ubuntu/neway-bot
   ExecStart=/home/ubuntu/neway-bot/venv/bin/python3 bot.py
   Restart=always
   RestartSec=5

   [Install]
   WantedBy=multi-user.target
   ```
   Затем:
   ```bash
   sudo systemctl daemon-reload
   sudo systemctl enable nevay-bot
   sudo systemctl start nevay-bot
   sudo systemctl status nevay-bot   # проверка, что всё запустилось
   ```
   Логи смотреть так: `sudo journalctl -u nevay-bot -f`

### 🥈 Альтернатива для совсем новичков (проще, но с компромиссами)

Если возиться с VPS не хочется, можно взять бесплатный тир хостинга
**fly.io** — он даёт небольшую бесплатную квоту, которой хватает на
одного лёгкого дискорд-бота, но требует привязку карты и иногда
меняет условия бесплатного тира, поэтому не гарантирован «навсегда».

Либо **Replit** с настройкой Always On — там Always On платный, так
что бесплатный Replit без доплат держать бота 24/7 не сможет (будет
выключаться при простое).

**Вывод:** если нужно по-настоящему бесплатно и навсегда — берите
Oracle Cloud Free Tier с systemd, как описано выше. Это самый честный
и надёжный бесплатный вариант на сегодня.

## Частые проблемы

**Бот не реагирует ни на одну команду.**
Проверьте, что вы пишете команду именно на сервере с ID
`1520321736611070013` — на других серверах бот молчит специально.

**Бот не видит сообщения / не отвечает вообще.**
Убедитесь, что включён **MESSAGE CONTENT INTENT** в Developer Portal
(см. Шаг 1).

**`-cancel` говорит «не хватает прав» при попытке снять роль.**
Роль бота должна стоять **выше** снимаемых ролей в списке ролей
сервера. См. раздел «⚠️ Важно про роль бота» выше.

**`-report` находит мало сообщений или не находит вовсе.**
Бот может читать историю только в тех каналах, куда у него есть
доступ (`View Channels` + `Read Message History`). Если пользователь
писал только в канале, закрытом от бота, его сообщения туда не попадут.

## Если что-то нужно поменять

Все ID ролей/каналов, цвета и тексты сообщений собраны в `config.py`
— большинство правок делается там, без изменения логики в `bot.py`.
Если нужно подправить что-то ещё (раздельные кулдауны на команды,
другой лимит сообщений, доп. роли с доступом и т.д.) — просто скажите,
поправим.
